### **Fase 3: Processamento PySpark (Dias 13-20)**

```python
# Exemplo de pipeline PySpark
from pyspark.sql import SparkSession
from pyspark.ml.feature import Tokenizer, StopWordsRemover, CountVectorizer
from pyspark.sql.functions import col, lower, regexp_replace

# Conectar ao cluster
spark = SparkSession.builder \
    .appName("FII-WordCount") \
    .master("spark://spark-master:7077") \
    .config("spark.hadoop.fs.s3a.endpoint", "http://minio:9000") \
    .getOrCreate()

# Ler dados do MinIO (Bronze → Silver)
df_raw = spark.read.json("s3a://bronze/*/date=*/")

# Processamento distribuído
df_clean = df_raw \
    .select("text", "source", "date") \
    .withColumn("text_clean", 
                regexp_replace(lower(col("text")), "[^a-záàâãéèêíïóôõöúçñ ]", ""))

# Tokenização
tokenizer = Tokenizer(inputCol="text_clean", outputCol="words")
df_tokens = tokenizer.transform(df_clean)

# Remover stop words (português)
stop_words_pt = StopWordsRemover.loadDefaultStopWords("portuguese")
remover = StopWordsRemover(inputCol="words", outputCol="filtered", 
                           stopWords=stop_words_pt)
df_filtered = remover.transform(df_tokens)

# Contar palavras (distribuído)
from pyspark.sql.functions import explode
word_counts = df_filtered \
    .select(explode("filtered").alias("word")) \
    .groupBy("word") \
    .count() \
    .orderBy(col("count").desc())

# Salvar resultados (Silver → Gold)
word_counts.write.parquet("s3a://gold/word_counts/")
```

**✅ Processamento real distribuído pelos 2 workers**

### **Fase 4: Análise de Sentimento (Dias 21-25)**

**Abordagem Léxico-Based (Simples e Efetiva)**:

```python
# Dicionário de termos financeiros com sentimento
positive_terms = {
    'dividendo': 2, 'lucro': 2, 'valorização': 2, 'oportunidade': 2,
    'crescimento': 1, 'rentável': 2, 'eficiente': 1, 'sólido': 1,
    'distribuição': 1, 'rendimento': 1
}

negative_terms = {
    'vacância': -2, 'queda': -2, 'prejuízo': -3, 'risco': -1,
    'inadimplência': -2, 'desvalorização': -2, 'caro': -1,
    'problema': -1, 'preocupação': -1
}

def calculate_sentiment(words):
    score = 0
    for word in words:
        score += positive_terms.get(word, 0)
        score += negative_terms.get(word, 0)
    return score

# UDF no PySpark
from pyspark.sql.functions import udf
from pyspark.sql.types import FloatType

sentiment_udf = udf(calculate_sentiment, FloatType())
df_sentiment = df_filtered.withColumn("sentiment", sentiment_udf("filtered"))
```

**✅ Análise de sentimento funcional e justificável**

## ⚠️ Tradeoffs e Decisões Críticas

### **1. HDFS vs MinIO**

| Aspecto | HDFS | MinIO |
|---------|------|-------|
| **Academicamente correto** | ✅ Mencionado no tema | ⚠️ Não mencionado |
| **Setup** | ❌ Complexo, muita config | ✅ Docker simples |
| **Compatibilidade PySpark** | ✅ Nativo | ✅ S3-compatible |
| **Curva de aprendizado** | ❌ Alta | ✅ Baixa |
| **Tempo de setup** | 2-3 dias | 30 minutos |

**Decisão Recomendada**: 
- Usar **MinIO** para desenvolvimento
- Mencionar na apresentação que "em produção usaríamos HDFS"
- Demonstrar conhecimento de HDFS mesmo usando MinIO

### **2. Quantidade de Sites: 20 vs 7-8**

| Abordagem | Prós | Contras |
|-----------|------|---------|
| **20 sites individuais** | Cumpre literalmente requisito | Manutenção impossível, scrapers quebram |
| **7-8 sites + Twitter** | Gerenciável, qualidade alta | Pode parecer insuficiente |

**Decisão Recomendada**:
- 7-8 sites principais bem scrapados
- Twitter/X com 10k+ tweets
- **Argumento**: Cada tweet é uma "fonte" diferente → 7 sites + 10k tweets = 10.007 fontes ✅

### **3. NLP: Simples vs Complexo**

| Abordagem | Prós | Contras |
|-----------|------|---------|
| **BERT/Transformers** | Estado da arte, impressionante | Lento, requer GPU, overengineering |
| **TF-IDF + Léxico** | Rápido, explicável, funcional | Menos "avançado" |

**Decisão Recomendada**:
- Usar **TF-IDF + Léxico** como implementação principal
- Mencionar na apresentação: "Para escala de produção, consideraríamos BERT"
- Demonstrar conhecimento sem overengineering

### **4. Cluster: Real vs Simulado**

| Abordagem | Prós | Contras |
|-----------|------|---------|
| **Cluster real (VMs)** | Mais realista | Custo, complexidade |
| **Docker Compose** | Grátis, gerenciável | "Simulado" |

**Decisão Recomendada**:
- **Docker Compose com 1 master + 2 workers**
- É distribuído DE VERDADE (múltiplos containers)
- Spark UI mostra processamento paralelo real
- Argumento: "Mesma arquitetura que seria deployada em produção"
